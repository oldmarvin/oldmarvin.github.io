在上篇中，博客基本已经出来了，接下来进行一定的美化工作。

将本地的Hexo文件更新到Github的库中

在上一篇中，已经将博客部署到了centos服务器。该部署出来的服务器，在chrome浏览器中可以正常显示，但是在360极速浏览器，IE等的显示不正常。这一点我至今没有找到原因。

为了试验这种情况，我试着将本地的Hexo文件更新到Github的库中。

关于SSHkey的配置，之前我在第二篇中已经说明了，接下来就是配置完Key后的操作，如果还没有配的话，请查阅博文《Centos系统下Hexo+github+next搭建个人博客二：git的基本操作》

1、登录Github打开自己的项目 oldmarvin.github.io

2、点击SSH，选择SSH类型地址，复制地址出来。

3、打开本地的_config.yml，找到deploy段进行修改。

deploy:
  type: git
  repository: https://github.com/oldmarvin/oldmarvin.github.io.git
  branch: master

4、在Hexo文件夹下执行

hexo g
hexo d

执行完之后会让你输入github的账号和密码，输入完后就可以登录我们自己的部署在Github Pages服务器上的博客了。对应的地址是 username.github.io(我的是：olamarvin.github.io)。

假如这时候，报错 ERROR Deployer not found: git，那么就是你的deployer没有安装成功，你需要执行如下命令再安装一次：

npm install hexo-deployer-git --save
hexo g -d

你的博客就部署到Github上了。

5、在浏览器上输入自己的主页地址https://oldmarvin.github.io

更换主题

进入Hexo的官网主题专栏,挑选喜欢的主题，然后进入到它的github地址，只要把这个地址复制下来，例如选择hexo-theme-next这个主题，这个主题也是使用率极高的主题。

克隆主题

cd /usr/local/oldmarvin.github.io
git clone https://github.com/theme-next/hexo-theme-next themes/next

修改Hexo配置文件

下载完成后，打开Hexo文件夹下的配置文件_config.yml，修改参数为：theme: hexo-theme-next

hexo g

如果效果满意，将它部署到Github上。

hexo clean   (必须要，不然有时因为缓存问题，服务器更新不了主题)

Centos系统下Hexo+github+next搭建个人博客三：hexo+nginx博客初现

Posted on 2019-05-22 | Edited on 2019-05-23

Hexo

Hexo 是一个快速、简洁且高效的博客框架。Hexo 使用 Markdown（或其他渲染引擎）解析文章，在几秒内，即可利用靓丽的主题生成静态网页。

安装Hexo

npm install hexo-cli -g
npm install hexo-deployer-git --save

第一句是安装hexo，第二句是安装hexo部署到git page的deployer，两个都需要安装。

查询是否安装成功：

hexo version

安装 Hexo 完成后，请执行下列命令，Hexo 将会在指定文件夹中新建所需要的文件。像我是在/usr/local创建文件夹

hexo init oldmarvin
cd oldmarvin

新建完成后，指定文件夹的目录如下：

├── _config.yml
├── package.json
├── scaffolds
├── source
|   ├── _drafts
|   └── _posts
└── themes
└── public

对这些文件说明如下：

1、_config.yml是YAML格式文件，也是Hexo的站点配置文件

2、package.json配置hexo运行需要的node.js包，不用手动更改

3、scaffolds是模板文件夹。这个“模板”就是指新建的markdown文件的模板，每新建一个markdown文件（由于Hexo使用markdown语法，在渲染生成静态HTML页面之前，源文件都是markdown文件），就会包含对应模板的内容。

该文件夹内有三个模板：

  • draft.md，草稿的模板
  • page.md，页面的模板
  • post.md，文章的模板

4、source是资源文件夹，资源文件夹是存放用户资源的地方。除posts 文件夹之外，开头命名为 (下划线)的文件 / 文件夹和隐藏的文件将会被忽略。Markdown 和 HTML 文件会被解析并放到 public 文件夹，而其他文件会被拷贝过去。

5、themes是主题文件夹。Hexo 会根据主题来生成静态页面。

6、public，该文件夹中的文件最终会被上传到github之中

几个最主要的命令

hexo g：hexo generate
hexo c：hexo clean
hexo s：hexo server
hexo d：hexo deploy

输入hexo generate命令来生成静态文件，生成的静态文件存放在public文件夹中

输入hexo server(可简写为hexo s)来启动服务器，默认情况下，端口号为4000，上文已经开启过此端口号，所以服务器启动成功后，在浏览器中输入http://<你的IP地址>:4000，即可看到第一次运行的状况： 按Ctrl+C则退出该运行服务器。

安装并配置Nginx

安装Nginx主要是想让网站能随时随地访问，不像上文中hexo提供的server，只能手动输入命令后才能访问到网站。

1、安装gcc

yum -y install gcc
gcc --version

2、安装g++

需要注意的是安装的命令是gcc-c++而不是g++

yum -y install gcc-c++
g++ --version

3、pcre安装

tar -zxvf pcre-8.42.tar.gz    
cd pcre-8.42/    
./configure    
make    
make install

4、zlib安装

tar -zxvf zlib-1.2.11.tar.gz    
cd zlib-1.2.11/    
./configure    
make    
make install

5、openssl安装

tar -zxvf openssl-1.1.0h.tar.gz    
cd openssl-1.1.0h/    
./config    
make    
make install

6、nginx安装

tar -zxvf nginx-1.14.0.tar.gz    
cd nginx-1.14.0/    
./configure --prefix=/usr/local/nginx --with-http_ssl_module --with-pcre=../pcre-8.42 --with-zlib=../zlib-1.2.11 --with-openssl=../openssl-1.1.0h
make    
make install

7、测试nginx是否安装成功

cd /usr/local/nginx/sbin
./nginx

浏览器访问如：http://47.106.35.83/（这是机器的IP地址），如果能正常显示nginx首页，则表示安装成功。

8、nginx的其他操作

我在实际中用到的nginx操作有这些：

#查找目录：
find / -name "nginx*"

#做好命令的转接
ln -s /usr/local/nginx/sbin/nginx /usr/bin/nginx

#修改nginx.conf，最主要是加上一句，这样就可以把其他系统的Nginx配置文件引入进来：
include /usr/local/nginx/conf/conf.d/*.conf;

#创建conf.d目录，并创建pj.conf文件
mkdir conf.d
touch pj.conf
vim pj.conf

#启动nginx
nginx -c /usr/local/nginx/conf/nginx.conf    

#重启nginx
./nginx -s reload 

#修改nginx最大上传文件限制：
#参考文章：https://www.cnblogs.com/xiaoliu66007/p/6397917.html
#增加这句话：
location /
{
client_max_body_size 1000M;
}

#停止nginx: 
ps –ef|grep nginx
kill -QUIT 2072
kill -QUIT 主进程号（上面ps命令看到的带master字样的进程号）

配置静态服务器访问路径

Nginx 需要配置静态资源的路径信息才能通过 url 正确访问到服务器上的静态资源。
即是要将HEXO生成的静态资源的路径放置到Nginx的访问路径

打开 Nginx 的默认配置文件 /etc/nginx/nginx.conf ，将默认的 root /usr/share/nginx/html 修改为: root /…//public （此处可能在此配置文件的42行，即为hexo初始化的文件夹）

修改完成后保存，输入以下命令重启Nginx：

nginx -s reload

此时再次访问你的IP地址，若显示上文的hexo初次运行的样子，则说明配置成功。

安装git

安装命令：

yum –y install git

查询是否成功命令：

git version

在github上注册账号

在github上新建一个名为 你的用户名.github.io 的仓库

像我的github用户名是oldmarvin，就新建 oldmarvin.github.io 的仓库，这里强调必须是按照这样的格式命名，其它名称无效），将来你的网站访问地址就是 http://oldmarvin.github.io 了。由此可见，每一个github账户最多只能创建一个这样可以直接使用域名访问的仓库。

配置sshkey，使git可以上传

配置Github的SSH密钥可以让本地git项目与远程的github建立联系，让我们在本地写了代码之后直接通过git操作就可以实现本地代码库与Github代码库同步。操作如下：

1、看看是否存在SSH密钥(keys)，在centos上运行

$ cd ~/. ssh

如果能够进入目录说明本机上已经有sshkey，否则需要创建一个

2、生成ssh-key，填入自己的邮箱

ssh-keygen -t rsa -C "xxxxxx@sina.com"

中间需要按三次回车确认。

3、在GitHub账户中添加你的公钥

运行如下命令，将公钥的内容复制出来。

cd ~/.ssh
ls
cat id_rsa.pub

如果你是在xshell中运行命令，可以按住 ctrl+insert 进行复制。

4、接着，登陆GitHub,进入你的Account Settings，选择SSH Keys，粘贴密钥，添加即可

5、测试是否配置成功

ssh -T git@github.com

这个命令不需要修改，直接打进去就好。

[root@iZuf60c5nb7ulqwzl2jwbsZ .ssh]# ssh -T git@github.com

RSA key fingerprint is MD5:16:27:ac:a5:76:28:2d:36:63:1b:56:4d:eb:df:a6:48.

中间需要键入一次yes，出现这个打招呼届面就是成功了。

基本的git操作

1、远程下载别人的git代码

git clone https://github.com/theme-next/hexo-theme-next themes/next

2、设置本地的git账号用户名

输入下面的代码进行个人信息的设置，把名称和邮箱替换成你自己的，名字根据自己的喜好自己取，而不是GitHub的昵称。

$ git config --global user.name "oldmarvin" //用户名
$ git config --global user.email  "marvincai@sina.com"//填写自己的邮箱

在本次试验之中，对于将hexo部署在gitpage中，只需要打入

hexo -d

系统会提示输入用户名与密码，这样输入完之后，就可以将代码上传。

3、git的基本操作

Git在未进行commit操作之前，存在三种状态：Untracked files，Changes not staged for commit及Changes to be committed，每种状态之间可以随意进行互相转换。

查看状态

git status

前进至：Changes to be committed

git add hello.c

后退至“Untracked files”文件状态

git rm --cached hello.c

修改文件更新到暂存区
git add hello.c

撤销修改
git checkout – hello.c

针对一个文件的修改你最终改完执行add，再commit
git add hello.c
git commit -m “add hello.c file”

针对多个文件，你可以执行 git add .（注意add 后边的符号 点）,在git commit -m “”。

git add . 
git commit -m "add all files"

修改只有add到暂存区，才会加入到commit，才会提交到分支。

先查看本地的分支：

git branch -a

创建一个属于自己的分支，并进行合并：

git checkout -b 分支名  origin/主分支名

提交到分支

　　 1）添加到暂存库：git add .

　　 2）可以通过查看状态看是否存储成功:git status -s

　　 3）提交代码: git commit -m ‘描述信息’

　　 4）提交到分支: git push origin 创建的子分支

但是，需要注意的是，在下一次准备修改代码之前需要先git pull，更新一下代码，避免发生冲突

Centos系统下Hexo+github+next搭建个人博客一：环境准备

目标是搭建一个个人使用的博客，在参考了很多路线之后，自己做了非常多的尝试。目前网络上的参考文章的基本可能情况有以下几种：

客户端选择：windows本地，centos直接做客户端

服务器端选择：centos做服务器，gitpage做服务器

本文是探索过程的记录，将自己的探索记录分为最主要的几个部分记录一下。

阿里云的centos设置

刚好我购买的服务器是阿里云，安装了centos系统。在阿里云的控制台进行一下设置，开放4000端口与80端口。具体步骤如下：

1、进入阿里云管理控制台，选择管理实例

2、点击本实例安全组，选择配置规则

3、点击右上角的添加安全组规则

4、依次将80端口和4000端口添加进去，如下图，注意IP地址的部分阿里云要求是0.0.0.0/0的格式。

安装Node.js

Node.js 是运行在服务端的 JavaScript, 是基于 Chrome JavaScript V8 引擎建立的平台。Hexo基于Node.js，所以安装Node.js是必须的，具体步骤如下：

下载安装包

wget https://nodejs.org/dist/v6.10.1/node-v6.10.1-linux-x64.tar.xz

解压缩改名放到/usr/local

tar xvJf node-v6.10.1-linux-x64.tar.xz
mv node-v6.10.1-linux-x64 /usr/local/node-v6

软链接到 /bin 目录

ln -s /usr/local/node-v6/bin/node /bin/node
ln -s /usr/local/node-v6/bin/npm /bin/npm

配置环境变量：将 /usr/local/node-v6/bin 目录添加到 $PATH 环境变量中可以方便地使用通过 npm 全局安装的第三方工具

echo 'export PATH=/usr/local/node-v6/bin:$PATH' >> /etc/profile

使环境变量生效

source /etc/profile

测试是否安装成功

node -v
npm -v

七牛云图片存储功能使用，使用页面与python编程上传

使用七牛云来存储文件

注册七牛云账号并上传文件

在网站https://www.qiniu.com/注册账号，并且进行实名验证，这样就可以获得10G的免费图片空间。

新建存储空间之后便能够使用了。

图片文件上传，在菜单上点击内容管理即可以上传，一般来说会将日期作为上传时候的路径，以便于区分。上传之后，可以点击外链来获取图片的链接地址。

Python进行图片上传

官方文档：https://developer.qiniu.com/kodo/sdk/1242/python

1、首先安装七牛云包

2、获取开发者 Access Key 和 Secret Key。这两个key在注册后会在个人中心提供给我们的。点击右上角的个人图标，进入个人空间获得

3、获取存储空间的名字bucket_name

4、域名：这个域名如果不指定的话，七牛会给我们一个测试域名，我们暂时可以用这个测试域名，像我的域名是：prud1ioef.bkt.clouddn.com，这个颜色配置要说一下，并不明显，一开始并不知道这是系统给我们留的域名。

5、进行python编程：

创建文件：qiniuupload.py。这里有一个小坑，创建的文件名不能是qiniu.py，否则会报一些错误。

from qiniu import Auth, put_file, etag
import qiniu.config

access_key = 's6uGI8WQZFOTNWr-qnH2************'#替换成自己的AK
secret_key = 'ttFRNka2XfEuGXe73ns7************'#替换成自己的SK
url = 'http://prud1ioef.bkt.clouddn.com'
bucket_name = 'marvin'

q = Auth(access_key, secret_key)
def qiniu_upload(key, localfile):
    token = q.upload_token(bucket_name, key, 3600)
    ret, info = qiniu.put_file(token, key, localfile)
    if ret:
        return '{0}/{1}'.format(url, ret['key'])
    else:
        raise UploadError('上传失败，请重试')
